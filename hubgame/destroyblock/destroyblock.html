Features


Function generateBlock
Create an array to store all possible block patterns. Once all generated blocks have been used (checked via the 'placed' variable), a new set of blocks will be automatically re-generated.

Function drawGrid , drawPlacedBlocks , drawBlocks






Function drawGrid
This function renders the background grid of the game. It iterates through each cell in the grid and, if a cell is empty (value === 0), it draws a light orange square to indicate a slot where a block can be placed



Function drawPlacedBlocks
This function renders the blocks that have already been placed on the grid. It scans for grid cells with a value of 1 and draws the corresponding image (img) stored in that cell at the correct grid coordinates based on the "i" and "j" axes.

Function drawBlocks
This function renders the available block choices (the 3 randomized blocks) that have not yet been placed by the player (!block.placed). It calls the drawPattern function to draw these block shapes at their designated positions on the right side of the screen.



Function drawPattern
This function renders the block shapes as images. It iterates through the matrix of a given block, and if it encounters a "1", it draws the blockImage at the calculated position.

Function canPlaceBlock
This function is used to verify if a block can be placed at a specific location. It checks whether the block exceeds the grid boundaries or overlaps with an existing block already on the grid. It returns true if the placement is valid.

Function placeBlock
This function records the block placement onto the grid. It changes the value of the targeted grid cells to 1 and stores the blockâ€™s image (img) in those cells for future rendering.

Function clearFullLines
The core mechanic of the game, this function checks for and clears completed rows or columns. If a row or column is entirely filled with blocks, the system clears those cells, awards points to the player, and adds extra time.

Developed a Modular Enemy Framework based on the Separation of Concerns principle. By decoupling AI logic, shared attributes, and visuals into distinct modules , I created a highly scalable system that allows for rapid integration of diverse enemy types. Leveraged Polymorphism to implement specialized behaviors like aerial pathing and status afflictions, while optimizing performance through Integer Hashing and efficient Data Structures
