Game Logic Overview



Features


generateBlock (Randomized Patterns)
Created an array system to store various block shapes. The algorithm tracks the 'placed' status of each block and automatically re-generates a new set of patterns once all current choices have been utilized.

Rendering & Coordinate System






drawGrid (Grid Visualization)
Renders the background layout by iterating through the grid array. If a cell value is 0 (empty), it displays a placeholder, establishing the visual coordinate system for the player.



drawPlacedBlocks (State Rendering)
Handles the rendering of blocks already locked into the grid. It scans the grid state for values of 1 and maps the corresponding image assets to their exact pixel coordinates.

canPlaceBlock (Boundary & Collision)
A core validation algorithm that checks if a block can be legally placed. It ensures that the block does not exceed grid boundaries and prevents overlapping with existing cells.

clearFullLines (Line Clearing Algorithm)
The primary game mechanic that scans both rows and columns. If a line is fully occupied (all values === 1), the algorithm clears those cells, updates the score, and resets the grid state.
